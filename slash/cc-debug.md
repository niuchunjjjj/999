我的代码已经跑起来了，但**行为不对**（比如：按钮点了没反应、数据显示错误、加载半天没动静、动画不流畅等）。这种情况看不到明显报错，需要主动调试。

请：

1. **先问我两件事**（如果我没说清）：
   - 当前现象：你看到的是什么？（点了什么 → 期望发生什么 → 实际发生什么）
   - 涉及的页面 / 组件 / 数据来源

2. **给我一份"看得见"的调试路径**，按下面这个顺序教我用 Chrome 开发者工具排查：

   - **打开 DevTools**：Mac 上 `⌘ + ⌥ + I`，Windows / Linux 上 `F12` 或 `Ctrl + Shift + I`
   - **Console 面板**：先看有没有红色错误 / 黄色警告，把异常输出贴回来给我
   - **Network 面板**：刷新页面，看接口请求是不是都正常返回 200，重点关注红色的失败请求
   - **Elements 面板**：检查 DOM 结构和 CSS 样式是否符合预期
   - **Sources 面板**：在可疑的代码行打断点（点行号），刷新或触发动作，逐步执行看变量变化

3. **针对我的具体问题**，告诉我：
   - 最有可能从哪个面板入手（不要让我盲查 4 个面板）
   - 在 Console 里运行什么命令可以快速验证（比如 `console.log(状态变量)`、`document.querySelector('xxx')`）
   - 如果是后端 / 接口问题，怎么用 Network 面板复制请求 cURL 自己验证

4. **教我怎么**把发现的关键信息（报错截图、Console 输出、Network 请求详情）整理出来——这些信息也是给我向 AI / 同事求助时最有用的"证据"。

记住：**先用人话告诉我"应该看哪里、看什么"**，再讲技术细节。零基础也要能跟着操作。
